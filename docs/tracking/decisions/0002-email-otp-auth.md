# 0002 — 이메일 OTP 인증 채택

## 상황

개인 앱으로 출발하되 나중에 다른 사람도 사용할 수 있어야 함. 비밀번호 관리(해싱, 분실 복구) 없이 인증을 구현하고 싶음.

## 결정

Resend를 통한 이메일 6자리 OTP(원타임 코드). 검증 후 JWT(30일) 발급.

## 검토한 대안

**비밀번호 인증**: bcrypt 해싱 + 비밀번호 분실 복구 흐름이 필요. 개인 앱에서 인증 로직 무게가 과함.

**OAuth(소셜 로그인)**: 소셜 계정 연동 복잡도가 높고, 외부 OAuth 제공자에 종속됨.

## 결과

- 이메일 접근이 없으면 로그인 불가 (이메일 서비스 장애 시 신규 로그인 불가, 기존 JWT는 유효).
- Resend API 키가 외부 의존성으로 추가됨.
- 현재 발송 횟수 제한 없어 이메일 스팸 위험 있음 — rate limiting 추가 예정 (findings F-4).
- JWT는 30일 유효, 서버 측 revocation 없음. 로그아웃은 클라이언트 localStorage 삭제로만 처리.
