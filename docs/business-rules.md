# 도메인 규칙

## 핵심 엔티티

- **사용자(User)**: 이메일로 식별. 최초 로그인 시 자동 생성(UPSERT).
- **책(Book)**: 사용자가 등록. 상태 생명주기를 가짐.
- **문장(Quote)**: 책에 종속. 사용자 소유. 책 삭제 시 함께 삭제(CASCADE).
- **컬렉션(Collection)**: 사용자가 만든 책 묶음. 동일 사용자 내 이름 유일.
- **OTP 토큰**: 6자리 숫자 코드. 10분 유효, 1회용.

---

## 책 상태 전환

책의 상태는 `'읽고 싶음'`, `'읽는 중'`, `'완독'` 세 가지다.

- **전환**: 어떤 상태에서든 다른 상태로 자유롭게 변경 가능.
- **`'완독'`으로 전환 시**: `reread_count`가 0이면 1로 계산하여 전송해야 함. (`Math.max(1, 현재값)`) — 이 계산은 클라이언트에서 수행하며 서버는 전달된 값을 그대로 저장한다.
- **`'완독'` 외 상태로 전환 시**: 클라이언트가 `reread_count=0`, `is_favorite=0`을 계산하여 함께 전송해야 함. 서버는 상태 전환을 감지하여 자동으로 리셋하지 않는다.

## 인생책 (is_favorite)

- `'완독'` 상태의 책에만 등록 가능.
- `'완독'` 외 상태로 전환 시 클라이언트가 `is_favorite=0`을 함께 전송하여 해제. 서버는 상태와 독립적으로 전달된 값을 그대로 저장한다.
- 등록하지 않으면 0, 등록하면 1 (boolean integer).
- 읽는 중이거나 읽고 싶은 책은 인생책으로 표시할 수 없음 — 이 제약은 클라이언트 UI에서만 강제됨.

## 회독 수 (reread_count)

- `'완독'` 상태에서만 의미 있음. 완독 시 최솟값 1.
- `'완독'` 외 상태일 때 UI에서 편집 불가 — 클라이언트가 0을 함께 전송. 서버는 값을 그대로 저장.
- 완독 이전에는 회독 수 개념 없음(0 저장).

## 별점 (rating)

- 1.0 이상 5.0 이하 실수. 미평가 시 null.
- 통계의 평균 별점(`avgRating`)은 null이 아닌 값만 집계, 소수점 1자리로 반올림.

---

## 문장

- 반드시 책에 속해야 함 (`book_id` 필수).
- `book_id`는 반드시 요청한 사용자의 책이어야 함. 다른 사용자의 책에는 문장 추가 불가(403).
- `text` 필수, 공백만인 경우 400.
- `page` 선택. 기록 시 양의 정수 (클라이언트 UI에서만 강제, 서버는 값 검증 없이 저장).

## 컬렉션

- 완독한 책(`status = '완독'`)만 컬렉션에 추가 가능.
  - 서버(`POST /api/collections/:id/books`): `book.status !== '완독'`이면 400 반환.
  - 클라이언트(`BookCard`): 완독 책에만 "컬렉션에 추가" 드롭다운 항목 표시.
  - `AddToCollectionModal`: 서버 400 수신 시 오류 메시지 표시.
- 동일 사용자 내 컬렉션 이름 중복 불가 (409 반환).
- 같은 책을 같은 컬렉션에 중복 추가해도 오류 없이 무시됨 (`ON CONFLICT DO NOTHING`).
- 컬렉션 삭제 시 해당 컬렉션의 책-컬렉션 연결도 함께 삭제됨 (CASCADE).
- 컬렉션 삭제가 책 자체에는 영향 없음.
